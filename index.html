What I do

Research

I dig deep to learn about user needs, workflows, and expectations. I talk directly to users and system stakeholders, and observe current-state workflows to get the full picture.

Analysis

I turn all my user research into workflow assessments and stakeholder needs. I list users’ tasks and potential use errors, then design risk mitigations with the cross-functional team. If it’s a medical device, I make sure it complies with IEC 62366 and the FDA guidance on usability engineering.

Design

I illustrate concepts from low- to high-fidelity screen design and interactive prototypes, iterating with users and the development team. I document the design in UI specifications. I make the occasional website too — I made this website template from scratch with HTML, CSS, and the Hugo static site generator.

Evaluation

Throughout design and development, I collect user feedback and observe user performance with the UX design early and often. Before launch (or regulatory submission), I test the system with users to make sure it’s safe and intuitive for use.
